Why Deepfake Protection Matters

Traditional face recognition is fundamentally broken. Modern AI can generate faces indistinguishable from reality.

Low Risk

🔬 How It Works

4-Layer Security Pipeline

Every authentication attempt passes through four independent verification modules. All must pass simultaneously.

👁️

Layer 1: Face Detection

OpenCV's SSD deep neural network with ResNet-10 backbone detects and localizes faces at 30+ FPS on CPU. Confidence threshold: 70%.

SSD + ResNet-10 ~15ms
🧬

Layer 2: Face Recognition

128-dimensional face encodings compared against stored user identity using L2 distance. Uses dlib with MediaPipe fallback.

128-D Encodings ~25ms
😊

Layer 3: Liveness Detection

Eye Aspect Ratio (EAR) blink counting and nose-tip head movement tracking using MediaPipe 468-point FaceMesh. Defeats all static attacks.

EAR + Head Tracking ~10ms
🤖

Layer 4: Deepfake Detection

Multi-signal fusion combining MesoNet CNN (50%), texture analysis (30%), and DCT frequency analysis (20%) for robust deepfake classification.

MesoNet + Texture + DCT ~45ms
🧠 Core Feature

Deepfake Detection — 3-Signal Fusion

Our deepfake detector uses three independent signals for robust classification. Even if one signal is fooled, the others catch the attack.

50%

🧠 MesoNet CNN

Meso4 architecture with 4 convolutional blocks (~28K parameters). Trained on the Kaggle "Deepfake and Real Images" dataset. Learns mesoscopic-level features that GANs cannot perfectly replicate.

ArchitectureConv2D(8) → Conv2D(8) → Conv2D(16) → Conv2D(16) → Dense(1)
Input Size256 × 256 × 3
OutputSigmoid (0=Fake, 1=Real)
ReferenceAfchar et al., IEEE WIFS 2018
30%

🔍 Texture Analysis

Classic computer vision techniques that analyze spatial texture properties. GAN-generated faces often show local inconsistencies in blur, sharpness, and edge density.

LaplacianBlur variance detection (oversmoothed faces = fake)
Block ConsistencyDivides face into regions, checks variance uniformity
Canny EdgesEdge density ratio (too few/many edges = suspicious)
20%

📡 Frequency (DCT) Analysis

2D Discrete Cosine Transform spectral analysis. GANs leave characteristic fingerprints in the frequency domain that are invisible to the human eye but detectable through spectral analysis.

Method2D DCT → log spectrum → band energy ratios
GAN ArtifactMissing high-frequency content (oversmoothing)
Periodic CheckPeak-to-median ratio for periodic artifacts
ReferenceDurall et al., CVPR 2020
